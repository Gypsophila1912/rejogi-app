# 技術スタック

---

## 0️⃣ 前提定義（最初に決める）

| 項目               | 内容                                                            |
| ------------------ | --------------------------------------------------------------- |
| プロダクト種別     | 学内文化祭向けレジ管理システム（イベント運用支援SaaS的構造）    |
| 想定ユーザー規模   | 同時接続 初期10人程度　将来配布後10～50人程度（文化祭運用想定） |
| 可用性目標         | 99%（学生開発前提の現実ライン）                                 |
| セキュリティ要件   | OAuth認証・RBAC権限制御                                         |
| パフォーマンス要件 | レジ操作レスポンス 1秒以内                                      |
| チーム体制         | 個人開発                                                        |
| リリース頻度       | 開発中は随時、運用は文化祭前に固定                              |
| 予算制約           | 無料枠運用前提                                                  |
| 既存資産           | React知識 / Web技術基盤盤                                       |

# 1️⃣ 技術スタック構成

---

## 🖥 フロントエンド

| 項目           | 採用技術        | 採用理由         | 評価観点           | 備考      |
| -------------- | --------------- | ---------------- | ------------------ | --------- |
| 言語           | TypeScript      | 型安全性・保守性 | 開発事故防止       | 必須      |
| フレームワーク | React           | エコシステム成熟 | 学習コスト低       | SPA構成   |
| UIライブラリ   | TailwindCSS     | 開発速度向上     | デザイン統一       |           |
| 状態管理       | Zustand         | 実装簡潔         | Server State分離   | Redux不要 |
| フォーム管理   | React Hook Form | 高速・軽量       | バリデーション性能 |           |
| テスト         | Vitest          | 軽量テスト可能   | CI適合性           |           |
